# PULSE ECHO 第1段階・役割分担型 v3

GitHub Pagesで動く、公開サイト＋役割分担型の管理画面です。

## 今回追加した考え方

管理画面の入口を次の4つに分けました。

1. 撮影する人
2. 編集する人
3. 確認する人
4. 全体管理者

原則として、撮影は出場チームの関係者が担当します。

撮影担当の責任範囲は、録画ボタンを押すことだけではありません。

- 撮影前の許可確認
- 横画面、三脚、画角の確認
- バッテリーと空き容量の確認
- 試し撮りと再生確認
- 前半・後半の保存
- 冒頭・中間・最後の再生確認
- 素材の共有先登録

までを一つの作業として扱います。

## PULSE ECHO側へ撮影を依頼された場合

撮影区分を「PULSE ECHO側へ撮影を依頼」にすると、次の条件が表示されます。

- 依頼チームが会場と対戦相手への確認を行う
- 依頼チームが現場確認者を1名出す
- 機材、会場、第三者、不可抗力による完全撮影は保証されない
- 撮影不備があれば、編集工程へ責任を移さず記録する

## 編集者

撮影素材の共有後に作業します。

- 編集担当者
- 編集中／完了／素材不備
- 前半・後半・ハイライトのYouTube URL
- スコア表示の確認
- 動画全体の確認

素材が無い、途中で停止している、破損している場合は
「素材不備で編集不可」として記録できます。

## 確認者

公開前に次を確認します。

- チーム名、日付、大会名
- スコア
- 前半・後半動画の再生
- 掲載・公開許可

すべて確認しないと「公開OK」にできません。

## 全体管理者

- 新しい試合の登録
- 撮影担当チームの設定
- 全試合の進捗確認
- 公開／公開停止
- バックアップ
- GitHub Pages用 data.js の書き出し

撮影確認、編集完了、最終確認OKが揃っていない試合は公開できません。

## 現在の制限

この版はGitHub Pagesだけで動くため、運営データはブラウザの localStorage に保存されます。
複数人のリアルタイム共有や動画アップロードはまだ行いません。

動画ファイル選択では、ファイル名と容量だけを確認します。
将来、三賢者が構築するバックエンドへ接続する想定です。

## 公開方法

フォルダ内のファイルをGitHubリポジトリへアップロードし、

Settings → Pages → Branch: main → Folder: / (root)

を選択します。

管理画面：

`https://ユーザー名.github.io/リポジトリ名/admin.html`

## 公開データの更新

1. 全体管理者画面で試合を登録
2. 撮影者が撮影確認と素材共有を保存
3. 編集者がYouTube URLを登録し編集完了
4. 確認者が公開OK
5. 全体管理者が「公開する」
6. 画面下の「公開用 data.js を書き出す」
7. GitHub上の data.js と入れ替えて Commit changes
